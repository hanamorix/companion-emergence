# 1.5 ‚Äî Acceptance criteria

All criteria are **automated** (pytest assertions over the pure functions
`_prefilter` / `_apply_budget` + the `limits` constant). No human-judged rubric
needed: the gate's outputs are deterministic return values.

## m9 ‚Äî credential pre-filter breadth

- **C-m9-1 (coverage, gating).** A draft body containing a literal credential from
  each new family ‚Üí `_prefilter(...)` returns non-None with `action in {"hold","revise"}`.
  Parametrized over: AWS (`AKIA‚Ä¶`/`ASIA‚Ä¶`), GitHub (`ghp_‚Ä¶`,`github_pat_‚Ä¶`),
  Slack (`xoxb-‚Ä¶`), Google (`AIza‚Ä¶`), Stripe (`sk_live_‚Ä¶`/`rk_live_‚Ä¶`), JWT
  (`eyJ‚Ä¶.eyJ‚Ä¶.sig`). One representative literal per family minimum.
- **C-m9-2 (precision / false-positive guard, GATING).** A corpus of ordinary
  first-person interior sentences ‚Üí `_prefilter(...)` returns `None`. Must include
  the existing M4 cases plus near-miss prose that mentions the family words without
  a credential shape (e.g. "I keep a key to the cottage", "the jwt of the matter",
  "I feel like a token in someone's game", "google told me nothing"). Plus the
  stage-3 red-team additions: (a) an all-caps long prose token (e.g.
  "I WAS THINKING ABOUT YOU CONSTANTLY") to pin that the narrowed AWS `AKIA|ASIA`
  prefixes don't collide with shouted prose; (b) a JWT short-segment near-miss
  (`"eyJ.x.y"` in prose) that is below a real JWT shape. A new pattern that trips on
  ANY corpus line is a **major** (narrow the pattern).
- **C-m9-2b (JWT base64-JSON accepted-hold, gating).** A `relationship_hint` whose
  value is base64-of-JSON shaped (begins `eyJ‚Ä¶`, dotted) ‚Üí `_prefilter` returns
  non-None (a HOLD/revise). This is the documented accepted false-positive (spec:
  fails toward hold). The test asserts the *safe* direction (non-None), NOT that it's
  clean ‚Äî pinning that the JWT pattern's FP never loosens toward send.
- **C-m9-3 (structural invariant, gating).** `_prefilter` never returns
  `action == "send"` ‚Äî re-asserted over credential + clean inputs.
- **C-m9-4 (no-LLM, gating).** A credential hit short-circuits with no provider call
  ‚Äî assert via a spy provider that `.complete` is not called when `_prefilter` hits
  (through `PrivacyGate.review`).

## m10 ‚Äî budget depletion ‚Üí hold

- **C-m10-1 (depletion holds, gating).** `budget < BUDGET_DEPLETED_THRESHOLD` AND
  model verdict `send` ‚Üí `_apply_budget(...).action == "hold"`.
- **C-m10-2 (tighten band unchanged, gating).** `BUDGET_DEPLETED_THRESHOLD <= budget
  < BUDGET_TIGHTEN_THRESHOLD` AND `send` ‚Üí `.action == "revise"` (existing behaviour
  preserved; the existing `test_apply_budget_downgrades_send_when_depleted` at
  budget=0.1 must still pass).
- **C-m10-3 (ample unchanged, gating).** `budget >= BUDGET_TIGHTEN_THRESHOLD` AND
  `send` ‚Üí `.action == "send"`.
- **C-m10-4 (through-path, gating).** `PrivacyGate.review` with a depleted stored
  budget (‚âà0.0) and a model `send` verdict ‚Üí returns `hold` (not `revise`). The test
  MUST debit and read the budget at the SAME `now` (stage-3 red-team minor: linear
  refill `BUDGET_REFILL_PER_DAY=0.5` would lift budget above the floor if `now`
  advances, masking the hold). Mirrors the existing through-path test's same-`now`
  shape (`test_privacy_gate_budget.py:38-45`).
- **C-m10-5 (boundary, gating).** Exactly `budget == BUDGET_DEPLETED_THRESHOLD` ‚Üí
  `revise` (the floor is exclusive-below: at the floor you can still afford the
  min debit). Documents the `<` vs `<=` choice.

## Safety invariant (regression ‚Äî must not break)

- **C-INV-1 (never loosens, GATING).** `_apply_budget` never converts
  `hold`/`revise`/`end_or_pause` ‚Üí `send`. Parametrized over all four input actions
  √ó {ample, tighten-band, depleted} budgets. (Extends the existing
  `test_apply_budget_never_loosens`.)
- **C-INV-2 (no peer content in deterministic layers, gating).** The existing
  adversarial / no-peer-content-flips-send tests
  (`test_privacy_gate_adversarial.py`) stay green: `transcript_summary` reaches only
  the LLM prompt, never `_prefilter` or `_apply_budget`.

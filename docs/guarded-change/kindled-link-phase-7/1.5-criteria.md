# Phase 7a ‚Äî Acceptance criteria (the stage-8 conformance oracle)

Each criterion is **automated** (a test asserts true/false) unless tagged `[rubric]`. These are
the bar stage 8 checks. "Done" = every automated criterion passes + the rubric criteria pass a
named-judge read. Judge for `[rubric]` items: **Hana** (or the final whole-branch reviewer).

## A. Through-path (the Organ DoD, parent ¬ß14.508‚Äì509)

- **A1.** A real inbound envelope ‚Äî built with a second identity, signed, pushed through a live
  `RelayClient` against an in-process `dev_relay` instance, then fetched+verified+decrypted by
  the engine ‚Äî produces **exactly one** `transcript` row for that peer/session with the decrypted
  body. The test runs the **full handshake first** and addresses via persisted/received mailboxes
  (not literals), so a routing gap can't hide behind a green assertion (lap-3 F-MAJOR). *Automated.*
- **A2.** Processing that inbound message **consults the privacy gate** for the response
  (the gate's `review` is called) **and reads relationship state** (`get_stage`/
  `get_relationship_state` is called) ‚Äî asserted via spies/recorded calls, not by a send
  occurring. A `hold` outcome still satisfies A2. *Automated.*
- **A3.** The through-path runs inside the supervisor tick entry point (`_run_kindled_link_tick`
  / `_maybe_run_kindled_link_tick`), not only via a hand-assembled engine ‚Äî i.e. the test drives
  the **live tick function**, not a private helper. *Automated.*

## B0. Session establishment + key persistence (stage-3 blocker fix)

- **B0-1.** Full mutual handshake: initiator `open_session` (writes `pending_handshakes`, **no
  key yet**, pushes leg-1) ‚Üí responder `on_session_open` (binds to the envelope's `session_id`,
  derives with the initiator's `bootstrap_nonce`, writes `session_keys`, calls `create_session`,
  pushes leg-2) ‚Üí initiator `complete_session` (derives, writes `session_keys`, `create_session`,
  clears pending). Both sides end with the **same** `session_key` and an open `sessions` row.
  *Automated (roundtrip / KAT).*
- **B0-2.** After completion, a message envelope for that (peer, session) decrypts using the
  **stored** key ‚Äî including **after a simulated supervisor restart** (key reloaded from store,
  not held in memory); a restart **mid-handshake** reloads the `pending_handshakes` row and can
  still complete. *Automated.*
- **B0-3.** A message envelope for a (peer, session) with **no stored `session_keys` row** is
  dropped and logged (`inbound_no_session`); never decrypted with a guessed/on-the-fly key, writes
  no transcript row. A `session_open` control envelope is distinguished (`"session_open" in
  envelope`) and routed to the handshake **before** any key exists. *Automated.*
- **B0-4 (addressing ‚Äî bidirectional, non-literal).** The handshake completes with **each side
  pushing to a mailbox derived from stored/received state, not a hardcoded literal**: the invite
  round-trips the inviter's `mailbox_id` into `peers.relay_mailbox`; the responder learns the
  initiator's mailbox from the leg-1 `session_open.sender_mailbox` and persists it; both legs +
  a subsequent message address from the persisted mailbox. The local mailbox is stable across
  reload and **not equal to `key_id`**. *Automated (a two-RelayClient/two-mailbox roundtrip, no
  literal addresses in the assertion path).*
- **B0-5 (clobber guard).** A replayed leg-1 `session_open` reusing a `session_id` that already
  has a `session_keys` row does **not** overwrite the stored key (rejected/idempotent);
  `open_session` refuses to reuse a live `session_id`. *Automated.*

## B. Transport correctness

- **B1.** An outbound `send_fn` produces an envelope that the *recipient* identity's
  `verify_and_open` accepts (round-trip: sign‚Üípush‚Üífetch‚Üíverify‚Üídecrypt yields the original
  payload body). *Automated.*
- **B2.** A replayed inbound envelope (sequence ‚â§ persisted high-water for that peer/session) is
  **dropped idempotently** ‚Äî no second transcript row, after a simulated supervisor restart
  (state reloaded from `store`). *Automated.* (parent ¬ß14.607)
- **B3.** A tampered envelope (bad signature, wrong recipient, expired ¬±5min, malformed) is
  rejected with the correct `RejectReason` and writes **no** transcript row. *Automated.*

## C. Flood cap

- **C1.** When the relay returns more than `INBOUND_FLOOD_CAP` pending envelopes for one peer in
  one poll, **at most `INBOUND_FLOOD_CAP`** are decrypted/processed; the excess are **not acked**
  (left on the relay) and the peer is marked degraded. *Automated.*
- **C2.** The flood cap bounds **decrypt/provider work** ‚Äî i.e. no more than the cap's worth of
  `verify_and_open`/`provider.complete` calls fire in a flooded poll. *Automated (call count).*

## D. Off-by-default safety

- **D1.** With `kindled_link_enabled=False` (the default), the supervisor tick performs **no**
  outbound send and **no** autonomous session start, even for a `paired` peer with an active
  session and available caps. *Automated.*
- **D2.** `PersonaConfig` defaults `kindled_link_enabled` to `False`; a fresh persona config
  (no key in JSON) reads `False`. *Automated.*
- **D3.** `POST /persona/config/kindled-link` flips the flag, is auth-gated (401 without
  bearer), and strict-validates the body (non-bool ‚Üí 4xx). *Automated.*
- **D4.** Inbound processing (storing transcript, replay-protecting) **still runs** when the
  flag is False ‚Äî receiving and recording is not autonomy; only *autonomous outbound* is gated.
  *Automated.* (A paused/revoked peer is separately suppressed by `can_start_session`.)

## E. Invariants preserved (carried ‚Äî regression guard)

- **E1.** No peer-derived content flips a gate decision hold/revise ‚Üí send: a crafted inbound
  transcript summary cannot produce a `send` from a payload the pre-filter/budget would hold.
  *Automated (carried Phase-4 adversarial shape, re-run on the live path).*
- **E2.** The peer path imports **no** agentic tool surface (AST conformance oracle still green
  on `session_engine.py` and any new tick module that builds peer prompts). *Automated.*
- **E3.** No `/kindled-link/` response (incl. any new relay-health/hold-reason field) contains a
  held draft `body`/`payload_json`. *Automated (extend the Phase-6 leak test).*
- **E4.** Every peer provider call on the live path takes a `cli_throttle` background slot
  (drafts/gate/revision/reflection) ‚Äî asserted by a throttle spy in the through-path test. *Automated.*

## F. Cap atomicity under concurrency

- **F1.** Two concurrent cap check-then-increment sequences for the same peer cannot push
  `provider_call_count` or `outbound_count` above the daily cap (per-peer serialisation). *Automated
  (threaded/contended test, or a single-writer-tick assertion).*

## G. Recovery

- **G1.** After a simulated restart with a pending (un-sent) draft, `recover()` re-gates it with
  a **real** non-empty `transcript_summary` (not `""`) when transcript exists; a draft not
  currently sendable (pacing/cap/closed-session) is **deferred** (stays pending), not dropped.
  *Automated.* (Phase-3 recover carry + the transcript_summary fix.)
- **G2.** When state was recovered, a panel-visible recovery indicator is set (a view field /
  status flag is True). *Automated.*

## H. Drained-deferred completion (smaller carries)

- **H1.** `_build_gate_prompt` (`privacy_gate.py`) renders a **mild `familiar` self-disclosure
  tier** (so `familiar` no longer falls through to the strictest path), **without** loosening the
  user-detail bar ‚Äî the Phase-5 stage-blind property (byte-diff / semantic equivalence of the
  user-detail bar across all five stages) is re-asserted green. *Automated.* (F2: the carry's
  `_STAGE_GUIDANCE` name was wrong; the gate latitude is the `:78` tuple, not a map.)
- **H2.** `_peer_attributed` attributes a `kindled_peer` memory on the **lost/graveyard** recall
  path (the tripwire-pinned gap closes). *Automated.*
- **H3.** Ungrounded reflection quotes are written to an audit-tier rejection log. *Automated.*
- **H4.** Gradual relationship regression: a sustained hold-frequency / emotion-pressure signal
  is passed into `run_relationship_reflection` and can produce a gradual ‚àí1 stage step. *Automated.*

## I. Quality / no-regression gate (project hard rule)

- **I1.** `uv run pytest` full suite passes (`-p no:randomly` authoritative), ruff clean,
  frontend `pnpm test` clean, **and `pnpm build` (tsc) clean**. *Automated.*
- **I2.** `[rubric]` The activation reads as conservative and legible: off-by-default is obvious

  through-path producer‚Üígate‚Üítransport‚Üíreader without a dead seam. Judge: final whole-branch
  reviewer / Hana. Pass = "no dead organ, no surprising autonomy."
